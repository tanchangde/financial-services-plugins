# 三表财务模型模板完成

完成并填充整合财务模型模板，确保损益表、资产负债表和现金流量表之间的正确链接。

## 模型结构

### 识别模板标签页组织

模板的标签页命名约定和组织方式各不相同。填充之前，请查看所有标签页以了解模板结构。以下是常见的标签页名称及其典型内容：

| 常见标签页名称 | 内容 |
|----------------|------|
| IS, P&L, Income Statement | 损益表 |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销表 |
| Debt, Debt Schedule | 债务明细表 |
| NOL, Tax, DTA | 净经营亏损表 |
| Assumptions, Inputs, Drivers | 假设输入和驱动因素 |
| Checks, Audit, Validation | 验证 |

**模板审查清单**
- 识别模板中存在哪些标签页（并非所有模板都包含每个明细表）
- 记录任何未列出的模板特定标签页
- 了解标签页之间的依赖关系（例如哪些明细表流入主报表）
- 定位每个标签页上的输入单元格与公式单元格

### 了解模板结构

在填充模板之前，熟悉其现有布局，以确保数据输入在正确位置且公式保持完整。

**识别行结构**
- 在每个标签页顶部找到模型标题
- 识别部分标题及其视觉分隔
- 找到表示单位的行（百万美元、%、倍等）
- 注意区分实际与预估期间的列标题
- 确认期间标签（例如FY2024A、FY2025E）
- 识别输入单元格与公式单元格（通常通过字体颜色区分）

**识别列结构**
- 确认最左列中的行项目标签
- 验证历史年份先于预测年份
- 注意分隔历史期间和预测期间的视觉边框
- 检查所有标签页之间的列顺序是否一致

**使用命名区域**
模板通常对关键输入和输出使用命名区域。输入数据前：
- 查看模板中的现有命名区域（Excel中的公式 → 名称管理器）
- 常见命名区域包括：收入增长率、成本百分比、关键输出（净利润、EBITDA、总债务、现金）、情景选择器单元格
- 确保输入进入流入这些命名区域的单元格

### 预测期间
- 模板通常从最后一个历史年份向前预测5年
- 验证历史（A）和预测（E）列清晰分隔
- 确认列使用财年标记（例如FY2024A、FY2025E）

## 利润率分析

**注意：以下利润率分析仅在用户提示或模板明确要求时才执行。如果没有提示，请跳过本节。**

在损益表（IS）标签页上计算并显示盈利能力利润率，以跟踪运营效率并便于同行比较。

### 需要包含的核心利润率

| 利润率 | 公式 | 衡量指标 |
|--------|------|----------|
| 毛利率 | 毛利润 / 收入 | 定价能力、生产效率 |
| EBITDA利润率 | EBITDA / 收入 | 核心运营盈利能力 |
| EBIT利润率 | EBIT / 收入 | 折旧摊销后的运营盈利能力 |
| 净利润率 | 净利润 / 收入 | 最终盈利能力 |

### 带利润率的损益表布局

在每个利润行项目下方直接显示利润率百分比：
- 毛利润下方显示毛利率 %
- EBIT下方显示EBIT利润率 %
- EBITDA下方显示EBITDA利润率 %
- 净利润下方显示净利润率 %

## 信贷指标

**注意：以下信贷分析仅在用户提示或模板明确要求时才执行。如果没有提示，请跳过本节。**

在资产负债表（BS）标签页上计算并显示信贷/杠杆指标，以评估财务健康状况、债务能力和契约合规性。

### 需要包含的核心信贷指标

| 指标 | 公式 | 衡量指标 |
|------|------|----------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | （总债务 - 现金）/ LTM EBITDA | 扣除现金后的杠杆 |
| 利息覆盖率 | EBITDA / 利息费用 | 偿债能力 |
| 债务 / 总资本 | 总债务 /（总债务 + 权益） | 资本结构 |
| 债务 / 权益 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | （流动资产 - 存货）/ 流动负债 | 即时流动性 |

### 信贷指标层级检查

验证乐观情景显示最强信贷状况：
- 杠杆：乐观 < 基准 < 悲观（越低越好）
- 覆盖率：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 契约合规跟踪

如果已知债务契约，添加将实际指标与契约阈值进行比较的明确合规检查。

## 情景分析（乐观 / 基准 / 悲观）

在假设标签页中使用情景切换（下拉菜单）和CHOOSE或INDEX/MATCH公式。

| 情景 | 描述 |
|------|------|
| 基准情景 | 管理层指引或共识预期 |
| 乐观情景 | 高于指引的增长、利润率扩张 |
| 悲观情景 | 低于趋势的增长、利润率压缩 |

**需要敏感化的关键驱动因素**：收入增长、毛利率、销售、一般及行政费用%、DSO/DIO/DPO、资本支出%、利率、税率。

**情景审计检查**：切换开关影响所有报表、BS在所有情景下平衡、现金勾稽、层级保持（乐观 > 基准 > 悲观对于净利润、EBITDA、自由现金流、利润率）。

## SEC申报数据提取

如果模板特别要求从SEC申报文件（10-K、10-Q）提取数据，请参阅 [references/sec-filings.md](references/sec-filings.md) 获取详细提取指南。此参考资料仅在用监管申报中的公开公司数据填充模板时才需要。

## 完成模型模板

本节提供完成任何三表财务模型模板的一般指导，同时保留现有公式并确保数据完整性。

### 第一步：分析模板结构

在输入任何数据之前，全面审查模板以了解其架构：

**识别输入与公式单元格**
- 寻找区分输入单元格和公式单元格的视觉提示（字体颜色、单元格底纹）
- 常见约定：蓝色字体 = 输入，黑色字体 = 公式，绿色字体 = 链接到其他工作表
- 使用Excel的追踪引用/从属引用（公式 → 追踪引用）了解单元格关系
- 检查可能控制关键输入的命名区域（公式 → 名称管理器）

**映射模板流程**
- 识别哪些标签页流入其他标签页（例如假设 → IS → BS → CF）
- 注意任何支持明细表及其与主报表的链接
- 在填充之前记录模板的特定行项目结构和结构

### 第二步：填充数据而不破坏公式

**数据录入黄金规则**

| 规则 | 描述 |
|------|------|
| 只编辑输入单元格 | 除非有意替换公式，否则永远不要覆盖包含公式的单元格 |
| 保留单元格引用 | 复制数据时，使用粘贴数值（Ctrl+Shift+V）以避免用源格式覆盖公式 |
| 匹配模板单位 | 输入数据前验证模板使用千、百万还是实际值 |
| 遵守符号约定 | 遵循模板的现有符号约定（例如费用为正数或负数） |
| 检查循环引用 | 如果模板使用迭代计算，确保启用迭代计算 |

**安全数据录入流程**
1. 识别指定用于输入的精确单元格（通常高亮或标记）
2. 首先输入历史数据，然后验证这些期间的公式是否正确计算
3. 输入用于预测计算的假设驱动因素
4. 检查计算输出以确认公式按预期工作
5. 如果必须修改公式单元格，请在更改之前记录原始公式

**处理预建公式**
- 如果公式引用您尚未填充的单元格，预期会出现临时错误（#REF!、#DIV/0!），直到所有输入完成为止
- 当公式产生意外结果时，追踪引用以识别缺失或错误的输入
- 在不检查所有标签页上的公式依赖关系的情况下，不要删除行/列

### 第三步：验证公式

**公式完整性检查**

在依赖模板输出之前，验证公式是否正确运行：

| 检查类型 | 方法 |
|----------|------|
| 追踪引用 | 选择公式单元格 → 公式 → 追踪引用，验证它引用正确的输入 |
| 追踪从属 | 验证关键输入流向预期的输出单元格 |
| 评估公式 | 使用公式 → 评估公式逐步检查复杂计算 |
| 检查硬编码 | 预测公式应引用假设，而不是包含硬编码值 |
| 用已知值测试 | 输入简单测试值以验证公式产生预期结果 |
| 跨表一致性 | 确保相同的公式逻辑适用于所有预测期间 |

**需要关注的常见公式问题**
- 混合绝对/相对引用导致跨期间复制时结果不正确
- 链接到外部文件或删除范围的链接断开（#REF!错误）
- 收入增长前早期期间的除零错误（#DIV/0!错误）
- 循环引用警告（可能是有意的，用于利息计算）
- 预测列之间的公式不一致（使用Ctrl+\查找差异）

**验证跨表链接**
- 确认出现在多个标签页上的值是链接的（不是重复的）
- 验证明细表总计与主报表上的相应行项目挂钩
- 检查所有标签页上的期间标签是否对齐

### 第四步：按标签页进行质量检查

在填充模板后，对每个标签页执行这些验证检查：

**损益表（IS）质量检查**
- 历史期间的收入数字与源数据匹配
- 所有费用行项目加总到报告的总数
- 小计（毛利润、EBIT、税前利润、净利润）正确计算
- 税务计算逻辑适当（正确处理亏损）
- 预测驱动因素引用假设标签页（无硬编码）
- 期间环比变化方向合理

**资产负债表（BS）质量检查**
- 资产 = 负债 + 权益（每个期间）（主要检查）
- 现金余额与现金流量表的期末现金匹配
- 营运资本账户与支持明细表挂钩（如适用）
- 留存收益正确结转：前期RE + 净利润 - 股息 +/- 调整 = 期末RE
- 债务余额与债务明细表挂钩（如适用）
- 所有资产负债表项目具有适当的符号（资产为正，大多数负债为正）

**现金流量表（CF）质量检查**
- 经营活动现金流顶部的净利润与损益表的净利润匹配
- 非现金添加项（折旧摊销、股票补偿等）与其来源明细表/报表挂钩
- 营运资本变化具有正确的符号（资产增加 = 现金使用 = 负数）
- 资本支出与PP&E明细表或固定资产滚动挂钩
- 融资活动与BS上的债务和权益账户变化挂钩
- 期末现金与资产负债表现金匹配
- 期初现金等于前期期末现金

**支持明细表质量检查**
- 期初余额等于前期期末余额
- 滚动逻辑完整（期初 + 添加 - 扣除 = 期末）
- 明细表总计与主报表行项目挂钩
- 计算中使用的假设与假设标签页匹配

### 第五步：跨报表完整性检查

验证单个标签页后，确认三张报表正确整合：

| 检查 | 公式 | 预期结果 |
|------|------|----------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金勾稽 | CF期末现金 - BS现金 | = 0 |
| 净利润链接 | IS净利润 - CF期初净利润 | = 0 |
| 留存收益 | 前期RE + 净利润 - 股息 - BS期末RE | = 0（根据需要为股票补偿/其他项目调整） |

### 第六步：最终审查

在认为模型完成之前：
- 切换所有情景（如适用）以验证每种情况下检查都通过
- 审查并解决或记录所有#REF!、#DIV/0!、#VALUE!和#NAME?错误
- 确认所有输入单元格都已填充（搜索占位符值）
- 验证所有标签页上的单位一致
- 在进行任何其他修改之前保存干净版本

## 模型验证和审计

本节整合了已完模板的所有验证检查和审计程序。

### 核心链接（必须始终成立）

请参阅 [references/formulas.md](references/formulas.md) 获取所有公式详情。

| 检查 | 公式 | 预期结果 |
|------|------|----------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金勾稽 | CF期末现金 - BS现金 | = 0 |
| 现金月度与年度 | 期末现金（月度）- 期末现金（年度） | = 0 |
| 净利润链接 | IS净利润 - CF期初净利润 | = 0 |
| 留存收益 | 前期RE + 净利润 + 股票补偿 - 股息 - BS期末RE | = 0 |
| 权益融资 | Δ普通股/资本公积（BS）- 权益发行（CFF） | = 0 |
| 第0年权益 | 第0年募集的权益 - 第1年期初权益资本 | = 0 |

### 符号约定参考

| 报表 | 项目 | 符号约定 |
|------|------|----------|
| CFO | 折旧摊销、股票补偿 | 正数（加回） |
| CFO | Δ应收账款（增加） | 负数（现金使用） |
| CFO | Δ应付账款（增加） | 正数（现金来源） |
| CFI | 资本支出 | 负数 |
| CFF | 债务发行 | 正数 |
| CFF | 债务偿还 | 负数 |
| CFF | 股息 | 负数 |

### 循环引用处理

利息费用产生循环性：利息 → 净利润 → 现金 → 债务余额 → 利息

在Excel中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。将最大迭代次数设置为100，最大更改设置为0.001。在假设标签页中添加电路断路器切换。

### 检查类别

**第一部分：货币一致性**
- 在假设中识别并记录货币
- 所有标签页使用一致的货币符号和比例
- 单位行匹配模型货币

**第二部分：资产负债表完整性**
- 资产 = 负债 + 权益（每个期间）
- 公式：资产 - 负债 - 权益（必须 = 0）

**第三部分：现金流量表完整性**
- 现金与BS挂钩（CF期末现金 = BS现金）
- 现金月度与年度：期末现金（月度）= 期末现金（年度）
- 净利润与IS挂钩（CF净利润 = IS净利润）
- 折旧摊销与明细表挂钩
- 股票补偿与IS挂钩
- Δ应收账款、Δ存货、Δ应付账款与营运资本明细表挂钩
- 资本支出与折旧明细表挂钩

**第四部分：留存收益**
- RE滚动检查：前期RE + 净利润 + 股票补偿 - 股息 = 期末RE
- 显示组件分解以便调试

**第五部分：营运资本**
- 应收账款、存货、应付账款与BS挂钩
- DSO、DIO、DPO合理性检查（如果在正常范围之外则标记）

**第六部分：债务明细表**
- 总债务与BS挂钩（流动 + 长期债务）
- 利息计算与IS挂钩

**第六部分b：权益融资**
- 权益发行收益与BS普通股/资本公积增加挂钩
- 权益带来的现金增加 = 权益账户增加（必须平衡）
- 权益发行勾稽：Δ普通股/资本公积（BS）= 权益发行（CFF）（必须 = 0）
- 第0年权益勾稽：第0年募集的权益 = 第1年期初权益资本

**第六部分c：净经营亏损明细表**
- 第1年/成立时的期初NOL = 0（新业务以零NOL开始）
- NOL仅在税前利润<0时增加（亏损必须已实现才能产生NOL）
- 递延税资产与BS挂钩（NOL明细表DTA = BS递延税资产）
- NOL利用 ≤ 税前利润的80%（2017年后的联邦限制）
- NOL余额为非负数（不能利用超过可用金额）
- NOL仅在税前利润<0时产生
- 应税收入 ≤ 0时税费 = 0

**第七部分：情景层级**
- 绝对指标：乐观 > 基准 > 悲观（净利润、EBITDA、自由现金流）
- 利润率：乐观 > 基准 > 悲观（毛利率%、EBITDA%、净利润%）
- 信贷指标：乐观 < 基准 < 悲观用于杠杆（反向）

**第八部分：公式完整性**
- 销售成本、销售、一般及行政费用、研发、股票补偿按收入百分比驱动（无硬编码）
- 预测年份公式一致
- 无#REF!、#DIV/0!、#VALUE!错误

**第九部分：信贷指标阈值**
- 根据契约阈值将指标标记为绿/黄/红
- 任何红旗摘要

### 主检查公式

将所有部分状态聚合到单个主检查中：
- 如果所有部分通过 → "✓ 所有检查通过"
- 如果任何部分失败 → "✗ 检测到错误 - 请参阅下方"

### 快速调试工作流程
