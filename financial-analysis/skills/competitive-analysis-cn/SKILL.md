# 竞争格局映射

---

## 关键标准 - 应用于每个分析

### 源文件优先原则

当提供源文件（Excel/CSV）时：
* **直接提取值** — 不执行自己的计算；使用出现的数字
* **保持一致性** — 当同一指标出现在多个位置时，确保整个过程使用相同的值
* **验证计算** — 如果提示要求您计算某些内容，验证您的结果与源文件中的相关数据匹配
* **仅按显示的四舍五入** — 使用与源文件相同的小数精度

### 提示忠实度

当提示指定精确要求时，严格遵循：

**幻灯片标题和部分名称：**
* 如果提示说"概览和竞争范围"幻灯片 — 完全使用该标题，而不是改述
* 如果提示说在"细分组合"部分内 — 完全使用"细分组合"作为部分标题
* 永远不要用创意替代品替换（例如当指定"概览和竞争范围"时不要使用"2024财年细分贡献分析"）

**图表vs表格：**
* 如果提示说"嵌入式图表" — 创建实际的PowerPoint图表对象，而不是表格
* 如果提示说"数据标签必须显示" — 这些放在图表元素上（条形图、切片、线），而不是表格单元格
* 表格和图表不可互换 — 使用指定的内容

**完整数据系列：**
* 如果提示列出7个竞争对手，包括全部7个 — 不是5个或6个
* 如果提示显示2015-2025年的数据，包括所有年份 — 不是子集
* 如果提示在图表中指定6个系列（Uber、Lyft、DiDi、Bolt、Grab、Gojek），包括全部6个 — 不是4个

**精确值和措辞：**
* 如果提示说`收入：438亿美元（+18%同比增长）` — 完全显示该格式
* 如果提示说"超过DoorDash 4:1，Lyft 8:1" — 使用那些确切比率，而不是"Lyft的7.6倍"
* 如果提示给出特定百分比（例如"Uber 30%，DiDi 35%"），使用完全相同的数字

**如有疑问：** 重新阅读提示。如果它明确指定某些内容，那不是建议——这是要求。

### 参考文件

此技能在`references/`文件夹中包含参考文件。按以下方式使用：
* **`references/schemas.md`** — 并购交易、情景分析和幻灯片结构的表格模板。在构建财务表或投资情景时参考。
* **`references/frameworks.md`** — 按行业的2x2矩阵轴对。在选择定位可视化维度时参考。

### 源质量层级

当来源冲突时，按此优先级排序：
1. **10-K/年报** — 经过审计，最高可靠性
2. **财报电话会议/投资者演示** — 管理层评论，前瞻性指导
3. **卖方研究** — 分析师估计，用于私人公司规模
4. **行业报告**（麦肯锡、Gartner等） — 市场规模，趋势
5. **新闻文章** — 仅用于最近的发展，验证主要来源

### 数据可比性
* **时间周期必须匹配** — 所有竞争对手指标来自同一财年。标记例外："（FY24）"vs"（2024年上半年）"
* **指标定义必须匹配** — 跨竞争对手使用相同的计算方法
* **货币标准化** — 为国际比较将所有数字转换为美元；注明使用的汇率和日期
* **缺失数据使用"-"** — 永远不要留空；对于私营公司，使用"N/A"或带"[E]"标记的估计
* **引用每个数字** — 格式："[公司] [文档]（[日期]）"
* **源文件忠实度** — 当提供Excel/CSV文件时，完全按给定使用值；不要重新计算或不同地四舍五入

### 设计和格式
* **幻灯片标题 = 洞察** — "规模领导者远离利基玩家"而不是"竞争分析"
* **幻灯片标题必须适合** — 一两行可以，但不要溢出；如需要减小字体（最小24pt）
* **路标 = 量化** — "利润率低于40%"而不是"利润率下降"
* **评级包括实际数据** — "●●● 1600亿美元"而不仅仅是"●●●"
* **需要幻灯片编号** — 每张幻灯片必须有页码

### 演示特定要求
* **需要实际嵌入式图表** — 饼图、条形图和折线图必须是真实的PowerPoint图表对象（通过pptx技能创建），而不是文本/ASCII表示
* **匹配提示结构** — 如果提示指定幻灯片结构，按其执行
* **竞争对手表格** — 对于全面分析：每个竞争对手的指标表+定性表。对于快速评估：单个组合表可接受。
* **细分财务** — 如有需要，包括收入和EBITDA。对于竞争对手或披露有限的情况，收入表可接受；注明"[EBITDA未披露]"

### 视觉参考

**查看`examples/Toast_Embedded_Payments_Competitive_Landscape_v3.pptx`并匹配其专业质量：**

**间距和溢出预防：**
- **标题与内容间隙** — 幻灯片标题底部和第一个内容元素之间最少0.4"
- **部分标题间隙** — 部分标题和下方内容之间最少0.25"
- **元素缓冲** — 任何两个元素（表格、文本框、图表）之间最少0.2"
- **页边距安全** — 所有内容距离幻灯片边缘至少0.5"
- **文本溢出** — 如果文本不适合，减小字体或拆分到多张幻灯片；永远不要让文本剪裁或重叠

**幻灯片标题：**
- **必须适合幻灯片宽度** — 一两行可以，但文本不能溢出或剪裁
- **如果标题太长** — 缩短措辞或减小字体（最小24pt）
- **将洞察放在前面** — 首先是关键点，其次是细节

**图表格式：**
- **图例在布局内** — 始终设置`include_in_layout=True`，这样图例不会与图表区域重叠
- **图例位置** — 饼图（≤6项）使用右侧，折线/条形图（≤4系列）使用底部
- **系列太多** — 如果>6个系列，考虑拆分为多个图表或改用表格
- **数据标签** — 对于饼图，在切片上显示百分比，而不是仅依靠图例

**字体（明确设置，永远不使用默认值）：**
- 幻灯片标题：28-32pt粗体
- 部分标题：16-20pt粗体
- 正文：11-14pt常规
- 表格文本：10-12pt常规
- 来源/脚注：8-10pt，灰色
- **一致规则**：deck中相同元素类型 = 相同字体大小

**布局：**
- 干净的网格对齐 — 表格和文本块对齐到一致的边距
- 慷慨的空白 — 不要拥挤幻灯片；让内容呼吸
- 视觉层次 — 最重要的洞察最大/最突出
- 每张幻灯片一个关键信息 — 下方是支持细节

**颜色：**
- 有限的调色板 — 最多2-3种颜色（一种强调色）
- 柔和色调 — 避免明亮/饱和颜色；使用海军蓝、灰色、柔和蓝
- **一致应用** — 整个过程中相同的颜色含义（例如强调色用于关键指标）

**表格：**
- 浅灰色表头行，带粗体文本
- 交替行阴影（微妙）或干净的白色，带细边框
- 数字右对齐，文本左对齐
- 足够的单元格填充 — 文本不应接触边框

**评级视觉：**
- ●●● / ●●○ / ●○○系统，配有实际指标
- 对比表格中放置一致

适应结构和方法以适合您的行业 — 但保持这种抛光水平。

## 工作流阶段

### 第一步：澄清要求
在开始之前，确认：
* **范围**：单公司深度研究或多公司比较？
* **输出**：演示或书面备忘录？
* **重点领域**：特定竞争对手、维度或战略问题？
* **投资背景**：需要情景/路标？
* **源文件**：提供哪些数据文件以及应提取哪些值？

### 第二步：研究 → 大纲 → 审核 → 创建
**在创建最终输出之前不要完成。**

在第二阶段执行以下10步分析工作流（步骤0-9）。在创建最终幻灯片或文档之前完成研究和大纲。

---

## 分析工作流

### 第0步：识别行业定义指标
在深入分析之前，确定该行业最重要的3-5个指标：

| 行业 | 关键指标 |
|------|----------|
| SaaS | ARR、NRR、CAC回收期、LTV/CAC、40法则 |
| 支付 | GPV、费率、附加率、交易利润率 |
| 市场 | GMV、费率、买家/卖家比率、复购率 |
| 零售 | 同店销售、库存周转、销售坪效 |
| 物流 | 产量、单位成本、准时交付%、产能利用率 |

对于未列出的行业，确定投资者和运营商用来衡量绩效的3-5个指标。

在所有竞争对手比较中一致使用这些指标。

### 第二步：市场背景
- 市场规模（当前和预测），附来源
- 增长驱动因素和阻力
- 重塑行业的关键趋势

**正确：**"嵌入式支付市场在2024年为800-1000亿美元，年增长率为20-25%（麦肯锡2024）"
**错误：**"市场很大且增长迅速"

### 第三步：行业经济学

映射价值流。方法因行业类型而异：
* **垂直结构** — 价值链层级，每层典型利润率
* **平台/网络** — 生态参与者及其之间的价值流
* **碎片化** — 规模化动态和利润率差异

### 第四步：目标公司资料

```
| 指标 | 值 |
|------|-------|
| 收入 | 49.6亿美元 |
| 增长 | +26%同比增长 |
| 毛利率 | 45% |
| 盈利能力 | 3.73亿美元调整后EBITDA |
| 客户 | 13.4万 |
| 留存率 | 92% |
| 市场份额 | ~15% |
```

**对于多细分公司，添加细分分解：**
```
| 细分 | 收入 | 收入同比增长 | 收入占比 | EBITDA | EBITDA同比增长 | 利润率 |
|------|------|-------------|---------|--------|---------------|--------|
| 细分A | 251亿 | +26% | 57% | 65亿 | +31% | 26% |
| 细分B | 138亿 | +31% | 31% | 25亿 | +64% | 18% |
| 细分C | 51亿 | -2% | 12% | -7.4亿 | -16% | -1% |
| 合计 | 440亿 | +18% | 100% | 65亿* | - | 15% |
```
*如适用，注明公司成本

### 第五步：竞争对手映射

使用适合的框架对竞争对手进行分组：
* **按商业模式** — 平台vs垂直vs水平
* **按细分** — 企业vs中小企业vs消费者
* **按姿态** — 直接vs相邻vs新兴
* **按来源** — 现有企业vs颠覆者vs新进入者

### 第六步：定位可视化

| 可视化 | 最适合 |
|--------|--------|
| 2x2矩阵 | 两个主导竞争因素 |
| 雷达/蜘蛛图 | 多因素比较 |
| 分层图 | 自然聚类/战略群体 |
| 价值链图 | 垂直行业 |
| 生态图 | 平台市场 |

### 第7步：竞争对手深度研究

**表1 — 指标：**
```
| 指标 | 值 |
|------|-------|
| 收入 | XX亿美元 |
| 增长 | +XX%同比增长 |
| 毛利率 | XX% |
| 市值 | XX亿美元 |
| 盈利能力 | XX亿EBITDA |
| 客户 | XX万 |
| 留存率 | XX% |
| 市场份额 | ~XX% |
```

**表2 — 定性：**
```
| 类别 | 评估 |
|------|------|
| 业务 | 他们做什么（1句话） |
| 优势 | 2-3点 |
| 劣势 | 2-3点 |
| 战略 | 当前优先事项 |
```

### 第八步：比较分析

```
| 维度 | 公司A | 公司B | 公司C |
|------|-------|-------|-------|
| 规模 | ●●● 1600亿 | ●●○ 450亿 | ●○○ 80亿 |
| 增长 | ●●○ +26% | ●●● +35% | ●●○ +22% |
| 利润率 | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 第九步：战略背景
- 并购交易（倍数、战略理由）
- 合作伙伴和整合趋势
- 融资模式
- 监管发展

### 第十步：综合

**竞争护城河评估：**
使用以下类别评估每个竞争对手的持久优势：

| 护城河类型 | 评估内容 |
|------------|----------|
| 网络效应 | 用户/供应商飞轮的强度；跨边vs同边效应 |
| 切换成本 | 技术集成深度、合同锁定、行为习惯 |
| 规模经济 | 产量下的单位成本；最低效率规模 |
| 无形资产 | 品牌价值、专有数据、监管许可证、专利 |

用支持证据将每个评为强/中/弱。

**必需的综合元素：**
- 持久优势（难以复制）— 映射到上述护城河类别
- 结构脆弱性（难以修复）
- 当前状态vs轨迹

**对于投资背景：**
```
| 情景 | 概率 | 关键驱动因素 |
|------|------|-------------|
| 乐观 | 30% | 市场份额提升、利润率扩张 |
| 基准 | 50% | 当前轨迹持续 |
| 悲观 | 20% | 竞争压力、利润率压缩 |
```
