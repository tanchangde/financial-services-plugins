# 投行材料检查器

对投资银行演示文稿进行全面的四个维度质量控制。

## 前提条件

检查前先提取演示文稿内容：
```bash
python -m markitdown presentation.pptx > content.md
```

要进行视觉检查，使用`pptx`技能工作流转换为图像。

## 检查工作流

### 1. 数据一致性

提取带幻灯片引用的数据：
```bash
python scripts/extract_numbers.py content.md --check
```

验证：
- 关键指标在所有幻灯片上一致（收入、EBITDA、倍数）
- 计算正确（总计、百分比、增长率）
- 单位一致（使用相同比例：百万vs十亿，% vs 基点）
- 单位格式一致（例如$M vs $MM，$B vs $Bn - 全程选择一种风格）
- 时间周期对齐（财年vs最近十二个月vs季度）

标记模式：
```
问题：收入不匹配
- 第3、8页：5亿美元
- 第15页：4.85亿美元（DCF输入）
操作：协调数据
```

### 2. 数据与叙事对齐

将声明映射到支持数据：
- 趋势声明 → 图表方向
- 市场地位声明 → 收入/份额数据
- 事实断言 → 验证准确性

标记矛盾：
```
问题：叙事与数据矛盾
- 第4页："利润率下降"
- 第7页图表：利润率18% → 22%
操作：更新叙事或验证数据
```

检查合理性（例如"1000亿美元市场中的第一名"但收入2亿美元 = 0.2%份额）。

### 3. 语言润色

扫描以下内容：
- 随意措辞（"非常好"、"很多"）
- 没有具体细节的模糊量词
- 缩写、感叹号
- 不一致的术语

请参阅[references/ib-terminology.md](references/ib-terminology.md)了解替换模式。

标记模式：
```
问题：随意语言（第12页）
- "这笔交易是板上钉钉的"
→ "这笔交易具有引人注目的价值主张"
```

### 4. 格式QC

审计每张幻灯片：
- **图表**：来源引用、坐标轴标签、图例
- **字体**：一致的字体、层级大小
- **数据**：一致的格式（1,000 vs 1K）
- **日期**：全程格式一致
- **脚注**：适当的来源和免责声明

## 输出

使用[references/report-format.md](references/report-format.md)中的模板呈现发现。

按严重程度分类：
- **关键**：数据不匹配、事实错误
- **重要**：语言、叙事对齐
- **次要**：格式不一致
