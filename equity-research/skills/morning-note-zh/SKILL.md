# 晨会笔记 (Morning Note)

## 工作流程

### 步骤 1：隔夜发展

扫描覆盖范围内的相关事件：

**财报与指引**
- 任何覆盖公司隔夜或盘前报告？
- 财报意外（收入、每股收益、关键指标的预期差）
- 指引变化（上调、下调、维持）

**新闻与事件**
- 并购公告或传闻
- 管理层变更
- 产品发布或监管决定
- 竞争对手分析师的升级/降级
- 影响行业的宏观数据或政策变化

**市场环境**
- 隔夜期货/盘前走势
- 行业 ETF 表现
- 相关商品或货币走势
- 今日关键经济数据发布

### 步骤 2：晨会笔记格式

保持紧凑——晨会笔记应 2 分钟内可读：

---

**[日期] 晨会笔记 — [分析师姓名]**
**[覆盖行业]**

**首要推荐：[标题——基金经理需要听的一件事]**
- 2-3 句关于关键发展及为何重要
- 股票影响：价格目标、评级重申/变化

**隔夜/盘前发展**
- [公司 A]：业绩/新闻的一句话摘要 + 我们的观点
- [公司 B]：业绩/新闻的一句话摘要 + 我们的观点
- [行业/宏观]：相关行业广泛发展

**今日关键事件**
- [时间]：[公司] 财报电话会议
- [时间]：经济数据发布（预期 vs. 我们的观点）
- [时间]：会议或投资者日

**交易思路**（如有）
- [多头/空头] [公司]：1-2 句投资论点 + 催化剂
- 风险：什么会使这个想法错误

---

### 步骤 3：财报快评

如覆盖公司报告，提供快速反应：

| 指标 | 一致预期 | 实际 | 预期差 |
|------|----------|------|--------|
| 收入 | | | |
| 每股收益 | | | |
| [关键指标] | | | |
| 指引 | | | |

**我们的观点**：2-3 句话——这对股票是好是坏？是否改变我们的投资论点？

**行动**：维持/升级/降级评级？调整价格目标？

### 步骤 4：输出

- 用于邮件/Slack 分发的 Markdown 文本
- 如需正式分发则用 Word 文档
- 保持最多 1 页——基金经理和交易员不会读更多

## 重要说明

- 要有观点——仅总结新闻而无观点的晨会笔记无用
- 最重要的事先行——不要埋没标题
- "无新闻"是有效的晨会笔记——说"隔夜无重大变化，维持配置"
- 区分可操作事件（财报、并购）和噪音（次要分析师笔记、非事件）
- 时间戳你的观点——如早上 6 点写，注明盘前可能在开盘时变化
- 如错了，在下次晨会笔记中承认——信誉比每次都正确更重要
