# 论点跟踪器 (Thesis Tracker)

## 工作流程

### 步骤 1：定义或加载论点

如创建新论点：
- **公司**：名称和股票代码
- **头寸**：多头或空头
- **论点陈述**：1-2 句核心论点（例如"多头 ACME——定价权 + 经营杠杆带来的利润率扩张，因组合转向软件"）
- **关键支柱**：3-5 个支持性论点
- **关键风险**：3-5 个将使论点无效的风险
- **催化剂**：可能证明/反驳论点的即将发生事件（财报、产品发布、监管决定）
- **目标价格/估值**：如论点实现价值多少
- **止损触发点**：什么会让你退出

如更新现有论点，询问用户新数据点或发展。

### 步骤 2：更新日志

对每个新数据点或发展：

- **日期**：何时发生
- **数据点**：什么变化（财报超预期、管理层离职、竞争对手行动等）
- **论点影响**：这是加强、削弱还是中和特定支柱？
- **行动**：无变化/增加头寸/减持/退出
- **更新后信心度**：高/中/低

### 步骤 3：论点记分卡

维护运行记分卡：

| 支柱 | 原始预期 | 当前状态 | 趋势 |
|------|----------|----------|------|
| 收入增长 >20% | 符合预期 | 第三季度 22% | 稳定 |
| 利润率扩张 | 落后 | 利润率同比持平 | 关注 |
| 新产品发布 | 待定 | 推迟至第二季度 | 观察 |

### 步骤 4：催化剂日历

跟踪即将发生的催化剂：

| 日期 | 事件 | 预期影响 | 备注 |
|------|------|----------|------|
| | | | |

### 步骤 5：输出

适用于以下的论点摘要：
- 晨会讨论
- 投资组合回顾
- 风险委员会演示

格式：简洁的 markdown 或 Word 文档，含记分卡、近期更新和当前信心度。

## 重要说明

- 论点应可被验证——如没有什么能否定它，那就不是论点
- 同等严格跟踪反面证据和正面证据
- 至少每季度回顾论点，即使没有重大变化
- 如用户管理多个头寸，提供完整投资组合论点回顾
- 以结构化格式存储论点数据，以便跨会话引用
