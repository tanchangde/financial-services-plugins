# 税收损失收割

## 工作流程

### 步骤 1：识别候选

扫描应税账户中带有未实现损失的头寸：

| 证券 | 资产类别 | 成本基础 | 当前价值 | 未实现损失 | 持有期 | 损失百分比 |
|----------|-----------|-----------|---------------|-----------------|---------------|--------|
| | | | | | 短期（ST）/ 长期（LT） | |

**优先级排序：**
1. 最大绝对损失（最大税务收益）
2. 短期损失优先（抵消按普通所得税率征税的短期收益）
3. 损失百分比最大的头寸（不太可能快速恢复）

### 步骤 2：收益/损失预算

计算客户的税务情况：

| 类别 | 金额 |
|----------|--------|
| 年初至今（YTD）已实现短期收益 | |
| 年初至今（YTD）已实现长期收益 | |
| 年初至今（YTD）已实现损失 | |
| 净收益/（损失）头寸 | |
| 从前几年结转的损失 | |
| **目标收割金额** | |

**税务节省估算：**
- 短期损失 × 边际普通所得税率
- 长期损失 × 资本利得税率
- 最多 3,000 美元净损失抵扣普通收入
- 超额部分结转

### 步骤 3：替代证券

为每个收割候选建议替代证券，该证券：
- 保持相似的市场敞口（相同资产类别、行业、地域）
- 非"实质相同"（洗售交易规则）
- 具有相似的风险/回报特征

| 卖出 | 替换为 | 理由 | 跟踪误差风险 |
|------|-------------|--------|-------------------|
| SPDR 标准普尔 500（SPY） | iShares 核心标准普尔 500（IVV） | 相同指数，不同基金公司 | 最小 |
| Vanguard 总国际（VXUS） | iShares MSCI ACWI 除美国（ACWX） | 相似敞口，不同指数 | 低 |
| 个股 ABC | 行业交易所交易基金（ETF）（XLK） | 更广泛的敞口，无洗售交易风险 | 中等 |

### 步骤 4：洗售交易检查

执行前验证无洗售交易：

- 检查家庭中所有账户（应税账户、个人退休账户（IRA）、Roth 账户、配偶账户）
- 30 天回溯：过去 30 天内是否购买了实质相同的证券？
- 30 天前瞻：30 天内禁止回购相同证券
- 检查可能导致洗售交易的股息再投资计划（DRIPs）
- 记录每笔交易的洗售交易窗口

| 卖出的证券 | 洗售交易窗口开始 | 窗口结束 | 股息再投资计划（DRIP）激活？ | 风险 |
|--------------|----------------------|-----------|-------------|------|
| | | | | |

### 步骤 5：执行计划

| 交易编号 | 账户 | 行动 | 证券 | 股数 | 估计收益 | 估计损失 | 替代证券 | 备注 |
|---------|---------|--------|----------|--------|--------------|-----------|-------------|-------|
| | | 卖出 | | | | | | |
| | | 买入 | | | | | | |

**摘要：**
- 估计收割损失总额：$
- 估计税务节省：$（边际税率 %）
- 净投资组合影响：最小（替代证券保持敞口）
- 洗售交易窗口管理：[日期]

### 步骤 6：收割后跟踪

30+ 天后，可选：
- 换回原始证券（如偏好）
- 维持替代证券（如无特殊原因换回）
- 更新成本基础记录
- 为税务报告记录文档

### 步骤 7：输出

- 收割机会清单（Excel）
- 交易执行表
- 洗售交易跟踪日历
- 税务节省估算摘要
- 替代证券理由

## 重要说明

- 洗售交易（wash sale）规则严格——违规将不允许扣除损失并调整成本基础
- 实质相同指相同证券，而非相同资产类别——跟踪不同指数的交易所交易基金（ETFs）通常可以
- 始终协调所有家庭账户，包括退休账户
- 考虑长期成本基础下调——收割会重置成本基础，这意味着未来更多收益
- 年末是收割旺季，但机会全年存在
- 12 月的共同基金资本利得分红可能产生额外的收割紧迫性
- 为税务报告和合规记录所有事项
- 并非所有损失都值得收割——交易成本和跟踪误差有实际成本
