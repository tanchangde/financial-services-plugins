# 尽职调查清单（Due Diligence Checklist）

## 工作流程

### 步骤 1：确定尽职调查范围

向用户询问以下信息：
- **目标公司**：名称、行业、商业模式
- **交易类型**：平台收购、附加收购（add-on）、成长型投资、资本重组（recap）、资产剥离（carve-out）
- **交易规模/复杂程度**：决定尽职调查深度
- **关键关注点**：任何需要优先处理的已知问题（客户集中度、监管、环境等）
- **时间表**：意向书（LOI）/ 交割的目标时间

### 步骤 2：生成各工作流清单

生成涵盖所有主要工作流的清单，根据行业定制：

**财务尽职调查**
- 盈利质量分析（QoE）— 收入和 EBITDA 调整
- 营运资本分析 — 正常化 vs. 实际
- 债务及类债务项目
- 资本支出（维护性 vs. 增长性）
- 税务结构和潜在风险
- 审计历史和会计政策
- 备考调整（常态化、协同效应）

**商业尽职调查**
- 市场规模和增长（TAM/SAM/SOM — 总可服务市场/可服务市场/可获得市场）
- 竞争定位和市场份额
- 客户分析 — 集中度、留存率、净推荐值（NPS）
- 定价能力和合同结构
- 销售管道和订单储备
- 上市策略（go-to-market）有效性

**法律尽职调查**
- 公司结构和组织架构图
- 重大合同（客户、供应商、合作伙伴）
- 诉讼历史和未决索赔
- 知识产权组合和保护
- 监管合规
- 雇佣协议和竞业禁止

**运营尽职调查**
- 管理团队评估
- 组织结构和关键人员风险
- IT 系统和基础设施
- 供应链和供应商依赖
- 设施和房地产
- 保险覆盖范围

**人力资源尽职调查**
- 组织架构图和员工数量趋势
- 薪酬对标分析
- 福利和养老金义务
- 关键员工留任风险
- 文化评估
- 工会/劳资协议

**IT/技术尽职调查**（适用于技术驱动型业务）
- 技术栈和架构
- 技术债务评估
- 网络安全状况
- 数据隐私合规（GDPR/CCPA/SOC2）
- 产品路线图和研发投入
- 可扩展性评估

**环境/ESG**（如适用）
- 环境负债
- 监管合规历史
- ESG 风险和机遇

### 步骤 3：状态跟踪

跟踪每个项目的状态：

| 项目 | 工作流 | 优先级 | 状态 | 负责人 | 备注 |
|------|-----------|----------|--------|-------|-------|
| QoE 报告 | 财务 | P0 | 待处理 | | |
| 客户访谈 | 商业 | P0 | 进行中 | | 10 个中已完成 3 个 |

状态选项：未开始 → 已请求 → 已收到 → 审核中 → 已完成 → 红旗事项

### 步骤 4：红旗事项汇总

维护尽职调查过程中发现的红旗事项清单：
- 发现的问题
- 所属工作流
- 严重程度（交易终止/重大/可控）
- 缓解措施或解决路径
- 对估值或交易条款的影响

### 步骤 5：输出

- Excel 工作簿，每个工作流一个标签页（默认）
- 汇总仪表盘：各工作流完成百分比、未完成项目、红旗事项
- 交易团队的周度状态更新格式

## 行业特定附加项

根据行业自动添加相关项目：
- **软件/SaaS**：ARR（年度经常性收入）质量、队列分析、托管成本、SOC2 合规认证
- **医疗保健**：监管审批、报销风险、支付方组合
- **工业**：设备状况、环境修复、安全记录
- **金融服务**：监管资本、合规历史、信贷质量
- **消费**：品牌健康度、渠道组合、季节性、库存管理

## 重要提示

- 优先处理影响 LOI 或交割的 P0 级项目
- 标记卖方响应缓慢的项目 — 可能表明存在问题
- 交叉引用虚拟数据室内容与清单以识别差距
- 随着尽职调查进展更新清单 — 这是一份动态文件
