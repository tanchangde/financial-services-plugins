# 项目筛选（Deal Screening）

description: 快速筛选流入的项目 — CIM（保密信息备忘录）、teaser（项目简介）和经纪商材料 — 根据基金的投资标准进行评估。提取关键交易指标，运行通过/失败框架，输出一页筛选备忘录。适用于审查新项目流、分类流入材料或决定是否进行首次通话。触发词："screen this deal"、"review this CIM"、"should we look at this"、"triage this teaser"或"deal screening"。

## 工作流程

### 步骤 1：提取交易事实

从提供的 CIM、teaser 或描述中提取：

- **公司**：名称、所在地、行业/子行业
- **描述**：业务内容（1-2 句话）
- **财务数据**：收入、EBITDA（息税折旧摊销前利润）、利润率、增长率
- **交易类型**：平台、附加收购（add-on）、资本重组（recap）、少数股权、资产剥离（carve-out）
- **要价/估值**：倍数、企业价值（如已说明）
- **卖方动机**：为什么现在出售
- **管理层**：继续持股还是退出
- **关键客户**：集中度风险
- **关键风险**：明显的红旗事项

### 步骤 2：根据标准筛选

应用基金的投资标准（如未知则询问用户）：

| 标准 | 目标 | 实际 | 通过/失败 |
|-----------|--------|--------|-----------|
| 收入范围 | | | |
| EBITDA 范围 | | | |
| EBITDA 利润率 | | | |
| 增长概况 | | | |
| 行业匹配度 | | | |
| 地理位置 | | | |
| 交易规模/企业价值 | | | |
| 估值（EBITDA 倍数） | | | |
| 客户集中度 | | | |
| 管理层连续性 | | | |

### 步骤 3：快速评估

提供三部分评估：

1. **结论**：通过 / 进一步尽职调查 / 直接否决
2. **看多理由**（2-3 条）：为什么这可能是一笔好交易
3. **看空理由**（2-3 条）：关键风险和担忧
4. **关键问题**：首次通话中需要解答的问题

### 步骤 4：输出

一页筛选备忘录，适合与合伙人或投资委员会（IC）快速分享。

## 重要提示

- 速度很重要 — 筛选应该在几分钟内完成，而不是几小时
- 直接指出红旗事项。不要掩盖担忧
- 如果财务数据看起来不一致或不完整，明确标记
- 如果是首次筛选，提前询问基金的标准
- 一旦确认，将筛选标准保存在记忆中以供未来交易使用
