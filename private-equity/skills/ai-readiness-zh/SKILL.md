# 投资组合 AI 准备度

## 工作流程

### 步骤 1：连接投资组合数据

首先，询问用户投资组合材料存放在哪里。不要假设——提供以下选项：

- **MCP 服务器** — 数据室、SharePoint、Google Drive，或已连接的投资组合运营数据库
- **本地文件** — 磁盘上的文件夹路径，包含季度演示文稿、财务数据、董事会材料
- **文件上传** — 直接将 PDF、PowerPoint 或 Excel 拖入对话

连接后，提取投资组合（或部分）的季度更新、董事会演示文稿和财务数据。对于每家公司，提取：行业、收入、各职能的员工人数、提及的技术栈，以及任何正在进行的 AI/自动化计划。

如果用户提供单一公司，仍运行扫描，但跳过跨投资组合排名。

如材料中不明显，请提前询问：
- 每家公司的剩余持有期（AI 投资回报在退出前 12 个月时重要性降低）
- 是否有任何 portco 已经部署了有效的 AI 方案

### 步骤 2：每家公司扫描

对于每家公司，回答三个门槛问题。三个都是"是" → **推进**。任何"否" → **暂缓**，并注明什么可以解锁。

1. **数据是否存在？** 他们能否为该用例提供干净的输入——客户列表、发票流、合同库——无需先做 6 个月的数据项目？
2. **是否有负责人？** 管理团队中是否有人会推动此事，而不是只是"支持"它的赞助人。
3. **能否在 30 天内试点？** 一个团队、一个工作流程、现成工具。如果答案以"首先我们需要……"开头，那就不是快速见效机会。

然后识别 2-3 个最高杠杆点。在成本结构和运营中寻找以下模式：

**后台办公室（通常试点最快）**
- 发票处理、应付/应收账款匹配、费用分类
- 合同摘要 — 供应商协议、租赁、客户 MSA
- 月末结账：对账、波动分析、贷款人报告初稿

**收入/前台**
- RFP 和提案初稿 — 如果收入是基于项目的，这是大杠杆
- 销售电话总结和 CRM 清理
- 客户支持工单分类和初响应起草
- 配置/复杂产品的报价

**运营（因行业而异）**
- SOP 和质量文档生成
- 排程和调度（现场服务、物流）
- 代码生成和审查（软件 portco）

对于每个杠杆点，用一句话说明：它替代了什么、每周节省的 FTE 小时数（假设 30-50%，而非 100%），以及它是现成购买还是需要轻度开发。

### 步骤 3：投资组合内排名

将每家公司的每个杠杆点整合到一个列表中。按以下因素排名：

1. **美元影响** — 年化 EBITDA 贡献（成本节约 + 收入提升，扣除工具成本）
2. **价值实现速度** — 到首次可衡量结果的月份
3. **概率** — 对数据质量、变革管理风险、管理团队能力的折现

平局决胜：优先考虑剩余持有期少于 18 个月的机会——那些需要立即行动或错失良机。

输出排名表：

| 排名 | 公司 | 机会 | 预估 EBITDA ($) | 价值实现月数 | 门槛 | 第一步 |
|---|---|---|---|---|---|---|
| 1 | | | | | 推进 | |
| 2 | | | | | 推进 | |
| 3 | | | | | 暂缓 — [阻碍因素] | |

### 步骤 4：寻找可复制的方案

在投资组合中运行一个成功方案是最高杠杆的举措。扫描以下情况：

- **同一行业、同一职能** — 两家医疗保健服务 portco 都有手动预授权？一次实施，两次部署。
- **同一工具、不同公司** — 如果一个 portco 已有有效的发票处理设置，标记所有其他 AP 金额超过 $Xm 的 portco 为快速跟进者。
- **共享供应商杠杆** — 三个 portco 购买同一工具是议价的机会。

列出每个可复制方案：领先公司（证明可行）和跟随公司（复制）。

### 步骤 5：输出

给运营合伙人一张纸，结构化用于投资组合审查：

1. **投资组合前 5** — 步骤 3 的排名表，包含负责人和 30 天第一步
2. **可复制方案** — 2-3 个可同时影响多家公司的方案
3. **每家公司的推进/暂缓** — 各一行；暂缓的，注明什么可以解锁
4. **我们不做的** — 那些看起来很好但未通过门槛的机会；为运营合伙人节省每个季度重新讨论的时间
5. **汇总 EBITDA 贡献** — 整个投资组合的 AI 机会总额，区分第一年快速见效 vs. 第 2-3 年规模化

## 重要提示

- **按美元排名，而非按兴奋度。** 一个在收入 $40m 公司节省 $400k 的无聊的 AP 自动化，永远胜过花哨的面向客户的聊天机器人。
- **约束条件几乎总是数据，而非模型。** 如果公司无法提供干净的客户列表，AI 不是第一个项目——而是数据清理。直说无妨。
- **优先现成方案。** 定制构建对没有工程深度的公司来说缓慢、昂贵且脆弱。优先选择他们可以购买和部署的工具。
- **所有权是真正的门槛。** 没有内部负责人的快速见效机会会在 90 天内夭折。如果管理团队中没有人想要它，无论金额大小，都标记为暂缓。
- **持有期决定紧迫性。** 距离退出还有 3 年的公司可以做基础数据项目。距离退出还有 12 个月的公司需要显示在 LTM EBITDA 中的东西给 CIM——否则就跳过。
- **失败的试点是信号。** 如果管理层已经尝试过某事但没有成功，在再次提出之前先找出原因。
