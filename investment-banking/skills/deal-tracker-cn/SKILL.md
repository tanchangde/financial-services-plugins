# 交易跟踪

description: 跟踪多个实时交易的里程碑、截止日期、行动项目和状态更新。维护交易管道视图并突出显示即将到来的截止日期和逾期项目。用于管理业务组合、跟踪流程里程碑或准备每周交易审查。触发词包括"交易跟踪器"、"交易状态"、"我们在哪里"、"流程更新"、"交易管道"或"每周交易审查"。

## 工作流程

### 第一步：交易设置

对每个交易，捕获：
- **交易名称/代号**：Project [Name]
- **客户**：卖方或买方名称
- **交易类型**：卖方、买方、融资、重组
- **角色**：牵头顾问、联合顾问、公平意见
- **交易规模**：预期企业价值
- **阶段**：委托前 → 已签约 → 营销 → IOI（兴趣指示函） → 尽职调查 → 最终投标 → 签署 → 关闭
- **团队**：分配的 MD、VP、Associate、Analyst
- **关键日期**：委托日期、CIM 分发、IOI 截止日期、管理层会议、最终投标截止日期、目标关闭

### 第三步：行动项目

在所有交易中维护运行中的行动项目列表：

| 行动 | 交易 | 负责人 | 截止日期 | 优先级 | 状态 |
|--------|------|-------|----------|----------|--------|
| | | | | P0/P1/P2 | 开放/完成/受阻 |

### 第四步：每周交易审查

为每周团队会议生成摘要：

**对于每个活跃交易：**
1. 一行状态更新
2. 本周关键进展
3. 即将到来的里程碑（未来 2 周）
4. 障碍或风险
5. 下周的行动项目

**项目储备摘要：**
- 按阶段划分的活跃交易总数
- 有风险的交易（错过的里程碑、停滞的流程）
- 管道中的新委托/投标
- 本季度预期关闭

### 第五步：输出

- Excel 工作簿包含：
  - 管道概述（所有交易，每行一个）
  - 每个交易的里程碑跟踪工作表
  - 行动项目主列表
  - 每周审查摘要
- 可选：用于电子邮件/Slack 分发的 Markdown 摘要

## 重要说明

- 每周至少更新跟踪表 — 过时的跟踪表不如不用
- 标记关键节点滑落的交易 — 早期预警可避免意外情况
- 无负责人和截止日期的行动事项无法落实 — 要具体
- 管道视图应显示交易阶段、规模和可能性 — 用于收入预测
- 记录买家/投资者反馈 — 反馈中的规律性意见可指导战略调整
- 单独归档已关闭/失败的交易 — 保持活跃视图干净
