## 工作流程决策树

**这是什么类型的任务？**

```
┌─ 用源数据填充空模板？
│  └─→ 遵循下面的"模板填充工作流程"
│
├─ 编辑现有的已填充幻灯片？
│  └─→ 提取当前内容、修改、重新验证
│
└─ 修复现有幻灯片上的格式问题？
   └─→ 参见"常见失败"表，应用针对性修复
```

## ⚠️ 关键渲染限制

**LibreOffice 用于验证但不能准确渲染 PowerPoint 文件。** 它可能导致字体、渐变、形状位置、文本换行和一些表格格式显示异常。

**这意味着什么：** 在 LibreOffice 中通过视觉验证的幻灯片可能在 Microsoft PowerPoint 中仍然有问题。验证循环捕获结构性问题（缺失内容、损坏的表格、保留占位符格式），但**不能**捕获字体替换、微妙的对齐偏移或渐变问题。

**必需操作：** 在交付输出时始终包括此声明：
> "此文件使用 LibreOffice 验证。请在分发前在 Microsoft PowerPoint 中审查，因为可能存在渲染差异。"

## 模板填充工作流程

复制并跟踪进度：

```
路演进度：
- [ ] 第 1 阶段：提取和验证源数据
- [ ] 第 2 阶段：将内容映射到模板部分
- [ ] 第 3 阶段：用适当格式填充幻灯片
- [ ] 第 4 阶段：验证 → 修复 → 重复直到干净
- [ ] 第 5 阶段：最终验证
```

### 第 1 阶段：数据提取
1. **在进行任何修改之前创建原始模板的备份** — 复制到 `[filename]_backup.pptx`。直接 XML 编辑或意外错误可能会损坏文件。
2. 识别所有源材料（Excel、CSV、PDF 报告、Word 文档、数据库、网络来源）
3. 从每个来源提取相关数据点
4. 针对原始来源验证所有数字
5. 标准化单位和货币（将所有数字转换为模板中使用的主要单位/货币）
6. 注意需要验证的任何计算 → 参见 [`calculation-standards.md`](reference/calculation-standards.md) 获取公式

### 第 2 阶段：内容映射
1. **打开并直观审查模板** — 在修改之前了解其结构、风格和现有内容
2. 分析模板结构 — 识别所有占位符区域和内容框
3. 将源数据映射到相应的模板部分 → 参见 [`slide-templates.md`](reference/slide-templates.md) 获取映射指导
4. 识别占位符指导框（任务创建者的彩色指令框）
5. 注意任何数据差距或不匹配 → 参见 [`slide-templates.md`](reference/slide-templates.md#handling-data-template-mismatches) 获取解决方案

### 第 3 阶段：模板填充
1. **删除或重新格式化占位符框** — 彩色指令框显示要创建什么，而不是如何格式。删除它们并在适当位置创建正确格式化的内容。参见 [关键反模式](#关键反模式 - 永远不要做这些)。
2. 用映射内容填充每个部分（首先关注内容）
3. **然后应用格式** 以匹配模板风格 → 参见 [`formatting-standards.md`](reference/formatting-standards.md)
4. 创建表格作为实际表格对象（永远不要使用管道/制表符分隔的文本）→ 参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)
5. 创建箭头/形状作为 PowerPoint 对象 → 参见 [`xml-reference.md`](reference/xml-reference.md#arrow-shapes)
6. 如果任务文件中提供了公司标志则插入；如果不可用，向用户标记："[未提供标志 - 请提供公司标志]"

### 第 4 阶段：验证 → 修复 → 重复

**这是一个反馈循环。重复直到所有检查通过或触发升级。**

```bash
# 转换为图像进行视觉验证
soffice --headless --convert-to pdf presentation.pptx
pdftoppm -jpeg -r 150 presentation.pdf slide
```

**验证清单（检查每张幻灯片图像）：**
- [ ] 文本在背景上可读？
- [ ] 表格是实际对象（列对齐，不是管道/制表符分隔的文本）？
- [ ] 图表/表格填充指定区域？
- [ ] 项目符号格式在部分内一致？
- [ ] 相同级别框的字体大小匹配？
- [ ] 没有内容超出幻灯片边界？
- [ ] **没有保留占位符格式**（没有填充数据的大彩色框）？
- [ ] **没有基于文本的"表格"**（没有 `|` 或制表符分隔符创建假列）？
- [ ] **跨幻灯片一致性**：所有出现的相同指标/数字在所有幻灯片上相同？

**修复循环协议：**

| 循环 | 操作 |
|-------|--------|
| 1 | 修复所有识别的问题，重新验证 |
| 2 | 修复剩余问题，重新验证 |
| 3 | 如果问题仍然存在，记录剩余问题并升级给用户 |

**3 个循环后，如果问题仍然存在：**
1. 列出每个未解决的问题，包括幻灯片编号和描述
2. 解释尝试了什么
3. 交付文件并附明确免责声明："以下问题无法自动解决：[列表]。需要手动审查。"

**不要** 无限期地继续循环。一些问题（字体渲染、复杂形状对齐）可能需要手动干预 PowerPoint。

### 第 5 阶段：最终验证

在交付之前运行 [最终质量清单](#最终质量清单)。

### 幻灯片层级级别（典型）

这些是典型范围 — 根据模板规范调整：

| 级别 | 示例 | 典型大小 | 风格 |
|-------|----------|--------------|-------|
| 标题 | 幻灯片标题 | 40-48pt | 粗体 |
| 副标题 | 市场定义、幻灯片说明 | 18-22pt | 粗体 |
| 部分标题 | "关键预测"、"评论" | 14-16pt | 常规 |
| 块标签 | "包括的细分"、"定义"侧边栏 | 12-14pt | 常规 |
| 块内容 | 要点、正文文本 | 11-14pt | 常规 |
| 表格标题 | 列标题 | 10-12pt | 粗体 |

| 脚注 | 来源、注释 | 8-9pt | 斜体 |

### 字体一致性匹配

**相同层级**的框必须使用相同的字体大小：

| 相同级别 | 必须匹配 |
|------------|-----------------|
| "包括的细分" | "排除的细分" |
| "定义" | "范围理由" |
| 左列项目符号 | 右列项目符号 |
| 所有块标签 | 彼此 |
| 所有部分标题 | 彼此 |

### 呈现的四舍五入

这些是**典型约定** — 根据值的大小和模板风格调整：

| 值类型 | 典型四舍五入 | 示例 |
|------------|------------------|---------|
| 大市场规模（100 亿美元+） | 最接近的 10 亿美元 | 185 亿 → 190 亿美元 |
| 小市场规模（<100 亿美元） | 最接近的 5 亿或 1 亿美元 | 23 亿 → 25 亿美元 |
| 大小范围 | 匹配来源的精度 | 149-221 亿 → 150-220 亿美元 |
| CAGR | 整数%或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 最接近的 5% 或匹配来源 | 21.4% → 20% |
| 倍数 | 1 位小数 | 9.69 → 9.7 倍 |

**原则：** 数值修约不应导致数值发生重大变化。对于较小的值，使用更精细的精度。

### 文本密度规则

- 每个内容框最多 6-7 个项目符号
- 每个项目符号最多 2 行
- 括号示例：同一行或缩进下方
- 无孤词（单个词在新行上）

### 对齐原则

**垂直堆叠的框** 必须有相同的：
- 左边距位置、项目符号缩进、文本开始位置、框宽度

**水平相邻的框** 必须有相同的：
- 顶部位置、高度（可能时）、内部填充

### 多幻灯片一致性

当相同数据出现在多张幻灯片上时：
- 使用相同的数字、格式和术语
- 如果一张幻灯片上的指标已更新，更新所有出现
- 验证期间交叉引用以捕获不匹配

## 必须要求

这些要求是不可协商的，无论模板如何：

| 要求 | 详情 |
|-------------|---------|
| **文本可读性** | 所有文本必须与背景有足够的对比度。示例：白色/浅色文本在深蓝色、深绿色、黑色背景上；黑色/深色文本在白色、浅灰色、浅黄色背景上。 |
| **实际表格对象** | 表格数据必须是表格对象，不是制表符分隔的文本。参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。 |
| **适当的图表/表格大小** | 粘贴的可视化必须填充指定区域。参见 [`formatting-standards.md`](reference/formatting-standards.md#proper-sizing-workflow)。 |
| **一致的格式** | 部分内的项目符号必须匹配（符号、大小、缩进）。相同级别的框必须使用相同的字体大小。 |
| **内容边界** | 所有内容必须保持在幻灯片边缘内。脚注框宽度：16:9 为~32.5cm，4:3 为~24cm。 |
| **无占位符格式** | 删除彩色指令框。主体：根据模板在浅色背景上的深色文本。 |

## 关键反模式 - 永远不要做这些

当占位符格式被误认为输出格式时会发生这些失败。识别这些模式至关重要。

### 反模式 1：将数据填充到占位符框中

**发生什么：** 模板有彩色指令框（黄色、橙色等）带有指导文本。模型用实际数据替换指导文本但保留彩色框。

**为什么错误：** 彩色框是占位说明框。它说明什么内容放在那里。输出应该有不同的格式 — 通常是白色/浅色背景上的深色文本，或适当风格的形状。

**识别测试：** 如果你的填充幻灯片有填充数据文本的大彩色矩形，你复制了占位符格式而不是替换它。

**关键区别 — 两种类型的"占位符"：**

| 类型 | 如何识别 | 做什么 |
|------|-----------------|------------|
| **指令框** | 鲜艳的颜色（黄色、橙色），包含指导文本如"在此插入 X"，彩色背景上的白色/浅色文本 | 删除整个形状，然后用生产格式创建新内容 |
| **布局占位符** | 幻灯片母版/布局的一部分，与模板主题匹配的中性颜色，"点击添加文本" | 保留形状，仅替换文本内容 |

如果不确定：检查形状是否存在于同一模板的空幻灯片上。布局占位符持续存在；指令框是常规形状。

### 反模式 2：基于文本的"表格"

**发生什么：** 模型使用分隔符（`|`、制表符、空格）创建类似表格的内容，而不是实际表格对象。

**为什么错误：** 这不是表格。列永远无法正确对齐，它无法一致地格式化，它看起来不专业。

**识别测试：** 如果你输入 `|` 字符或依赖空格/制表符创建列，你创建的是文本，不是表格。

**必须验证：** 创建任何表格后，验证它是实际表格对象。参见 [`xml-reference.md`](reference/xml-reference.md#critical-verify-tables-are-actual-table-objects) 获取验证方法。

### 反模式 3：继承占位符对比度

**发生什么：** 占位符使用彩色背景上的浅色文本（例如黄色上的白色）。模型填充数据但保持此配色方案，导致难以阅读的输出。

**为什么错误：** 占位符颜色故意不同以信号"替换我"。生产幻灯片通常在主体内容区域使用浅色背景上的深色文本。

**识别测试：** 如果你的填充内容在主体区域（不是标题）的彩色背景上有浅色/白色文本，你继承了占位符格式。

**正确方法：** 应用生产格式 — 通常主体内容使用白色或浅色背景上的深色文本（#000000 或 #333333）。标题和强调区域可以使用品牌颜色。

### 摘要：占位符 vs. 生产

| 元素 | 占位符（输入） | 生产（输出） |
|---------|---------------------|---------------------|
| 指令框 | 彩色背景、指导文本 | 删除或重新格式化 |
| 数据区域 | "[在此插入数据]" 文本 | 具有干净格式的实际数据 |
| 表格 | 表格应包含什么的描述 | 具有行/列的实际表格对象 |
| 正文文本 | 彩色背景上的浅色文本 | 浅色背景上的深色文本 |

**占位说明框说明需要填充什么内容，而非规定格式。**

## 错误处理

**如果 PDF/图像转换失败：**
1. 检查 LibreOffice 已安装：`which soffice`
2. 尝试替代：`libreoffice --headless --convert-to pdf presentation.pptx`
3. 如果仍然失败，在 PowerPoint/LibreOffice 中手动打开并导出

**如果源数据有不一致或冲突：**
1. **优先级顺序**：首先使用任务文件中明确提供的数据
2. 如果使用其他来源的数据（网络搜索、外部文档），向用户标记
3. 明确记录任何差异
4. 添加脚注解释数据源选择

**如果计算与源预测不匹配：**
1. 显示您的计算方法
2. 注意差异和可能原因（不同基年、方法）
3. 如果重大差异则呈现两个值
4. 向用户标记以解决

## 表格结构指南

创建表格时（必须是实际表格对象）：

**列对齐：**
- 文本列：左对齐（标题和内容）
- 数字列：右对齐或居中对齐（标题匹配内容）

**标题行：**
- 粗体文本
- 阴影背景（模板的品牌颜色）
- 白色或对比色文本

**共识/总计行：**
- 粗体文本
- 上方分隔线
- 独特的背景阴影

**宽度：** 完全填充指定部分宽度。

有关 XML 实现，参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。

## 脚注格式

**格式：**
```
来源：[来源 1]（年份），[来源 2]（年份）。
注释：(1) [第一个注释]；(2) [第二个注释]。
```

**示例：**
```
来源：Grand View Research（2024），Mordor Intelligence（2024），Markets and Markets（2023）。
注释：(1) 不包括硬件收入；(2) 包括 B2B 和 B2C 细分。
```

幻灯片正文中的所有上标数字（¹, ², ³）必须有相应的注释条目。

## 最终质量清单

在交付填充的模板之前，验证：

### 数据准确性
- [ ] 所有数字与原始源文档匹配
- [ ] 根据公式验证计算值（参见 [`calculation-standards.md`](reference/calculation-standards.md)）
- [ ] 年份和时间段正确
- [ ] 公司/竞争对手名称拼写正确
