## 关键验证公式

### 年复合增长率 (CAGR) 预测

**公式：**
```
未来价值 = 现值 × (1 + CAGR)^n
```

**变量：**
- 现值：当前/基准年市场规模
- CAGR：年复合增长率（小数形式，如 16.4% = 0.164）
- n：基准年与目标年之间的年数

**验证示例：**
```
源数据声称：22.1 亿美元（2024 年），按 16.4% 年复合增长率 = 55.0 亿美元（2030 年）

验证：22.1 × (1.164)^6 = 22.1 × 2.488 = 55.0 ✓
```

**计算 n（年数）：** 计算基准年与目标年之间的年数。示例：2024→2030 = 6 年，2025→2030 = 5 年。

### 估值倍数

**企业价值/收入 (EV/Revenue)：**
```
EV/Revenue 倍数 = 企业价值 ÷ 收入
隐含 EV = 收入 × 倍数
```

**企业价值/息税折旧摊销前利润 (EV/EBITDA)：**
```
EV/EBITDA 倍数 = 企业价值 ÷ EBITDA
隐含 EV = EBITDA × 倍数
```

**验证示例：**
```
源数据声称：4.36 亿美元交易，9.7 倍收入倍数，4500 万美元收入

验证：436 ÷ 45 = 9.69 ≈ 9.7x ✓
```

### 市场份额

**公式：**
```
市场份额 = (细分市场规模 ÷ 总市场规模) × 100
```

**验证示例：**
```
源数据声称：在线细分市场（18 亿美元）占总市场的 28%（65 亿美元）

验证：18 ÷ 65 = 0.277 = 27.7% ≈ 28% ✓
```

### 增长率

**同比 (YoY)：**
```
同比增长 = (当年 - 上年) ÷ 上年 × 100
```

**端点计算的 CAGR：**
```
CAGR = (终值 ÷ 初值)^(1/n) - 1
```

## 共识方法论

当源数据包含多个估计时，验证共识计算：

### 规模共识（范围）

**方法：** 所有来源的最小 - 最大全范围

**示例：**
```
来源：14.9 亿美元、18.3 亿美元、21.1 亿美元、21.2 亿美元、22.1 亿美元
共识：15-22 亿美元（四舍五入到最近的 1 亿美元）
```

### CAGR 共识（中心集群）

**方法：** 排除异常值（最高和最低），使用中心集群范围

**示例：**
```
来源：10.6%、16.4%、17.2%、19.0%、22.7%
排除异常值：10.6%（低）、22.7%（高）
中心集群：16.4%、17.2%、19.0%
共识：16-19% 或 16-17%（保守）
```

### 预测共识

**方法：** 将共识 CAGR 应用于规模范围的中点

**示例：**
```
规模范围：15-22 亿美元 → 中点：18.5 亿美元
CAGR 共识：16-17%
按 16%：18.5 × (1.16)^6 = 45.1 亿美元
按 17%：18.5 × (1.17)^6 = 47.5 亿美元
共识预测：45-48 亿美元
```

## 四舍五入指南

以下是**典型惯例**——根据数值大小和模板风格调整：

| 值类型 | 典型四舍五入 | 示例 |
|------------|------------------|---------|
| 大型市场规模（100 亿美元以上） | 最近的 10 亿美元 | 18.47 → 18 亿美元 |
| 较小市场规模（100 亿美元以下） | 最近的 5 亿美元或 1 亿美元 | 2.3 → 2.5 亿美元 |
| 规模范围 | 与源数据精度匹配 | 14.9-22.1 → 15-22 亿美元 |
| CAGR | 整数百分比或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 最近的 5% 或与源数据匹配 | 27.7% → 25% 或 30% |
| 收入（百万美元） | 1 位小数 | 18.47 → 1850 万美元 |
| 倍数 | 1 位小数 | 9.688 → 9.7x |

**四舍五入原则：**
- 四舍五入不应实质性改变数字——对于较小值，使用更精细的精度
- 一致性比精度更重要——对相似数字使用相同的四舍五入
- 创建范围时，低端向下舍入，高端向上舍入
- 对于汇总统计（均值、中位数），与输入数据的精度匹配

## 验证清单

在使用源数据中的任何计算值之前：

### 公式验证
- [ ] 预测使用正确的 CAGR 公式：`PV × (1 + r)^n`
- [ ] 倍数计算为 EV ÷ 指标（非反向）
- [ ] 增长率在分母中使用正确的基准年
- [ ] 百分比份额在适用情况下总和约为 100%

### 输入验证
- [ ] 基准年数字与源文档匹配
- [ ] CAGR/增长率与声明的源方法论匹配
- [ ] 时间段 (n) 计算正确
- [ ] 货币和单位一致（十亿美元 vs 百万美元）

### 输出验证
- [ ] 计算结果与源数据声明的数字匹配
- [ ] 如果不匹配，调查方法论差异
- [ ] 四舍五入一致应用
- [ ] 结果合理（无数量级错误）

### 共识验证
- [ ] 所有来源均包含在范围计算中
- [ ] 异常值排除方法论有文档记录
- [ ] 中点计算使用正确的平均
- [ ] 范围边界代表实际最小/最大值或文档记录的子集

## 需要调查的警示信号

**预测不匹配：**
- 计算预测与源数据差异 >5%
- 可能原因：基准年不同、CAGR 不同或四舍五入

**倍数不匹配：**
- 计算倍数与源数据不同
- 可能原因：指标定义不同（LTM vs NTM、收入 vs 净收入）

**共识不匹配：**
- 您的共识与源数据的共识不同
- 可能原因：源数据排除了某些数据点、异常值处理不同

**如有疑问：** 在脚注中注明差异并展示您的计算方法论。
