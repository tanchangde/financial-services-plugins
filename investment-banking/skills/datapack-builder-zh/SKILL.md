# 财务数据包构建器

为私募股权、投资银行和资产管理构建专业、标准化的财务数据包。将来自 CIM、发售备忘录、SEC 申报文件、网络搜索或 MCP 服务器访问的财务数据转换为适合投资委员会审查的精良 Excel 工作簿。

**重要：** 在此工作流程的整个过程中，对所有 Excel 文件创建和操作使用 xlsx 技能。

## 关键成功因素

每个数据包必须达到这些标准。任何一项不达标都将导致交付成果无法使用。

### 1. 数据准确性（零差错要求）
- 追踪每个数字到源文件并注明页码
- 专门使用基于公式的计算（无硬编码值）
- 交叉检查小计和总数的内部一致性
- 验证资产负债表平衡：资产 = 负债 + 权益
- 确认现金流量与资产负债表变化挂钩

### 2. 基本规则

**规则 1：财务数据（衡量金钱）→ 带 $ 的货币格式**
触发词：收入、销售、收入、EBITDA（息税折旧摊销前利润）、利润、亏损、成本、费用、现金、债务、资产、负债、权益、资本支出
格式：$#,##0.0 表示百万，$#,##0 表示千
负数：$(123.0) 而非 -$123

**规则 2：运营数据（计量实物）→ 数字格式，无 $**
触发词：单位、门店、地点、员工、客户、平方英尺、物业、人数
格式：#,##0 带逗号
负数：(123) 与表格其余部分一致

**规则 3：百分比（比率和比例）→ 百分比格式**
触发词：利润率、增长、比率、百分比、收益率、回报、利用率、入住率
格式：0.0% 表示一位小数
显示：15.0% 而非 0.15

**规则 4：年份 → 文本格式防止逗号插入**
格式：文本或自定义防止 2,024
显示：2020、2021、2022、2023A、2024E

**规则 5：当上下文混合时，每个指标获得其自己的适当格式**
示例：
```
细分分析，2022、2023、2024
零售收入，$50.0、$55.0、$60.0
  门店，100、110、120
  单店收入，$0.5、$0.5、$0.5
```
收入和单店指标使用 $，门店数量使用数字格式。

**规则 6：对所有计算使用公式 → 永远不要硬编码计算值**
所有小计、总数、比率和衍生指标必须基于公式，而非硬编码值。这确保准确性并允许动态更新。

### 3. 专业呈现标准

**格式标准：**

**配色方案 - 两层：**

**第一层：字体颜色（xlsx 技能强制）**
- **蓝色文本（RGB: 0,0,255）**：所有硬编码输入（历史数据、假设），而非普通文本
- **黑色文本（RGB: 0,0,0）**：所有公式和计算
- **绿色文本（RGB: 0,128,0）**：链接到其他工作表

**第二层：填充颜色（可选用于增强呈现）**
- 填充颜色是可选的，仅在用户请求或增强呈现时使用
- 如果用户请求颜色或专业格式，使用此标准方案：
  - **部分标题**：深蓝色（RGB: 68,114,196）背景配白色文本
  - **子标题/列标题**：浅蓝色（RGB: 217,225,242）背景配黑色文本
  - **输入单元格**：浅绿色/奶油色（RGB: 226,239,218）背景配蓝色文本
  - **计算单元格**：白色背景配黑色文本
- 如果指定，用户可以覆盖自定义品牌颜色

**两层如何一起工作（如果使用填充颜色）：**
- 输入单元格：蓝色文本 + 浅绿色填充 = "用户输入数据"
- 公式单元格：黑色文本 + 白色背景 = "计算值"
- 工作表链接：绿色文本 + 白色背景 = "来自另一个标签页的引用"

**字体颜色告诉你它是什么。填充颜色告诉你它在哪里（如果使用）。**

**重要：** xlsx 技能的字体颜色是强制性的。填充颜色是可选的 - 默认是白色/无填充，除非用户请求增强格式或颜色。

**始终应用：**
- 粗体标题，左对齐
- 数字右对齐
- 子项目 2 空格缩进
- 小计上方单下划线
- 最终总数下方双下划线
- 冻结行/列标题上的窗格
- 最小边框（仅在结构需要时）
- 一致字体（通常 Calibri 或 Arial 11pt）

**永远不包括：**
- 每个单元格周围的边框
- 多种字体或字体大小
- 图表，除非特别请求
- 过度格式或装饰

## 结构一致性

除非明确另有指示，否则使用标准 8 工作表结构：
1. 执行摘要
2. 历史财务（损益表）
3. 资产负债表
4. 现金流量表
5. 运营指标
6. 物业/细分表现（如适用）
7. 市场分析
8. 投资亮点

### 工作表 1：执行摘要
目的：为忙碌的高管提供一页概述

内容：
- 公司概述（2-3 句关于商业模式）
- 关键投资亮点（3-5 个要点）
- 财务快照表（过去 3 年 + 预测的收入、EBITDA、增长）
- 如适用的交易概述
- 显著显示的关键指标

格式：干净、粗体标题、最小装饰、强调关键数字

### 工作表 2：历史财务（损益表）
目的：完整的损益表历史

内容：
- 按细分/产品线划分的收入细分
- 销售成本/收入成本
- 毛利润和毛利率 %
- 运营费用明细（S&M、R&D、G&A）
- EBITDA 和调整后 EBITDA
- 线以下项目（D&A、利息、税项）
- 净利润

格式：
- 年份作为列（文本格式：2020、2021、2022）
- $ 百万或 $ 千（在顶部明确说明单位）
- 所有财务数据的会计格式
- 小计上方单下划线，净利润下方双下划线
- 所有数字右对齐

### 工作表 3：资产负债表
目的：期末财务状况

内容：
- 流动资产（现金、应收账款、存货、预付、其他）
- 长期资产（PP&E、无形资产、商誉、其他）
- 流动负债（应付账款、应计费用、债务流动部分、其他）
- 长期负债（长期债务、递延税项、其他）
- 股东权益（普通股、留存收益、其他）

格式：
- 验证公式：资产 = 负债 + 权益
- 一致的日期标记
- 包括营运资本计算
- 主要小计上方单下划线，最终总数下方双下划线

### 工作表 4：现金流量表
目的：现金产生和使用分析

内容：
- 经营现金流（间接法首选）
- 投资现金流（资本支出、收购、资产出售）
- 融资现金流（债务发行/偿还、权益、股息）
- 现金净变化
- 期初和期末现金余额

格式：
- 在可能情况下链接到损益表和资产负债表
- 显示净利润与经营现金流的调节
- 清晰标记现金使用（流出）与来源（流入）

### 工作表 5：运营指标
目的：非财务 KPI 和运营数据

内容（行业特定）：
- 单位销量、客户数量、地点
- 生产力指标（人均收入、单店收入、单单位收入）
- 产能利用率
- 市场份额
- 客户留存/流失率
- 行业特定 KPI

**关键格式说明：**
运营指标上没有美元符号。这些是数量，而非货币。

格式：
- 明确说明单位（客户、员工、门店、平方英尺等）
- 带逗号的整数：1,250 而非 $1,250
- 比率的百分比：95.0%
- 数字右对齐

### 工作表 6：物业/细分表现（如适用）
目的：按业务单元、物业或细分详细细分

内容：
- 按细分划分的收入和盈利能力
- 按地点/产品划分的关键指标
- 细分特定 KPI
- 比较表现分析

格式：与财务工作表一致，用于收入/EBITDA，数字格式用于运营指标

### 工作表 7：市场分析
目的：行业背景和竞争定位

内容：
- 市场规模和增长趋势
- 竞争格局概述
- 市场份额分析
- 行业基准和同行比较
- 监管环境（如相关）

格式：叙述文本和表格的混合，为市场数据引用来源

### 工作表 8：投资亮点
目的：关键投资论点要点的叙述摘要

内容：
- 竞争优势的详细撰写
- 增长机会和战略举措
- 风险因素和缓解策略
- 管理层评估和业绩记录
- 投资论点摘要

格式：清晰标题、要点、简洁段落

## 分步工作流程

### 第一阶段：文档处理和数据提取

**步骤 1.1：分析源数据**
- 访问源材料：上传的文档、公开申报文件的网络搜索或 MCP 服务器数据
- 审查数据结构并识别关键部分
- 定位财务报表（通常 3-5 年历史）
- 识别管理层预测（如包含）
- 注意财年结束日期
- 立即标记任何数据质量问题

**步骤 1.2：提取财务报表**
- 定位历史损益表数据
- 提取资产负债表快照（年末或季末）
- 查找现金流量表
- 提取管理层预测（如有）
- 注意所有页码以进行可追溯性

**步骤 1.3：提取运营指标**
- 识别与行业相关的非财务 KPI
- 捕获单位经济数据
- 提取客户/地点/容量数据
- 记录增长指标和趋势

**步骤 1.4：提取市场和行业数据**
- 竞争定位信息
- 市场规模和增长率
- 行业基准数据
- 同行比较信息

**步骤 1.5：注意关键背景**
- 交易结构和理由
- 管理团队背景
- 源材料中的投资亮点
- 风险因素和考虑事项
- 任何数据差距或不一致

### 第二阶段：数据归一化和标准化

**步骤 2.1：归一化会计呈现**
- 确保所有年份的行项目名称一致
- 标准化收入确认处理
- 识别并记录一次性费用
- 如有需要创建"调整后 EBITDA"调节
- 注意任何会计政策变化

**步骤 2.2：应用格式检测逻辑**
对于每个数据点，根据完整上下文确定格式：
- 读取工作表名称、表格标题、列标题和行标签
- 应用基本规则（见上文）
- 不确定时，检查原始源文档
- 默认使用更干净的格式（简洁优先）

**步骤 2.3：识别归一化调整**
常见调整记录：
- 重组费用（如果真正非经常性则加回）
- 股票薪酬（根据行业标准加回）
- 收购相关费用（加回，说明金额）
- 法律和解或诉讼费用（评估复发风险）
- 资产出售或减值（从经营业绩中排除）
- 关联方调整（归一化到市场利率）
注意：源引用格式因数据源而异（文档的页码、网络的 URL、MCP 数据的服务器引用）

**步骤 2.4：创建调整时间表**
对于每个归一化：
- 记录调整了什么及原因
- 引用来源（文档页码、URL 或数据源引用）
- 按年量化美元影响
- 评估复发风险
- 显示从报告到调整后数字的计算

**步骤 2.5：验证数据完整性**
- 确认小计使用公式正确求和
- 验证资产负债表平衡
- 检查现金流量与资产负债表变化挂钩
- 跨工作表交叉检查数字一致性
- 标记任何差异以供调查

### 第三阶段：构建 Excel 工作簿

**关键：对所有 Excel 文件操作使用 xlsx 技能。进行之前阅读 xlsx 技能文档。**

**步骤 3.1：创建标准工作表结构**
创建包含以下工作表的工作簿：
- 执行摘要
- 历史财务
- 资产负债表
- 现金流量
- 运营指标
- 物业表现（如适用）
- 市场分析
- 投资亮点

**步骤 3.2：构建每个工作表并应用适当格式**
系统地应用格式规则：
- 标题：粗体、左对齐、11pt 字体
- 财务数据：货币格式 $#,##0.0 表示百万
- 运营数据：数字格式 #,##0（无 $）
- 百分比：0.0% 格式
- 年份：文本格式防止逗号插入
- 负数：使用带括号的会计格式
- 下划线：小计上方单线，总数下方双线

**步骤 3.3：插入公式进行计算**
- 所有小计和总数必须基于公式
- 在适当情况下将资产负债表链接到损益表
- 将现金流量链接到损益表和资产负债表
- 创建跨工作表引用进行验证
- 避免硬编码任何计算值

（正确模式和常见错误的详细代码示例 - 保持与英文版本相同）

**步骤 3.4：应用专业呈现**
- 在每个数据工作表上冻结顶行和第一列
- 设置适当的列宽（通常 12-15 个字符）
- 所有数字数据右对齐
- 所有文本和标题左对齐
- 根据会计标准添加单/双下划线
- 确保干净、最小化的外观

### 第四阶段：情景构建（如包含预测）

**管理层情景：**
按源材料中提供的呈现公司预测：
- 提取所有管理层假设
- 记录增长率、利润率扩张、资本需求
- 标记任何需要审慎对待的急剧增长拐点
- 清晰标记为"管理层情景"

**基准情景（风险调整）：**
根据公司特定风险因素对管理层预测应用保守调整：
- 应用反映执行风险和历史预测准确性的收入增长折扣
- 根据行业基准和运营杠杆调节利润率扩张假设
- 如果依赖增长则增加资本支出假设
- 如果低估则增加营运资本需求
- 如适用，根据整合复杂性延迟协同效应实现
- 记录所有调整及理由和支持分析

**悲观情景（LBO 分析建议但可选）：**
基于行业周期性和公司脆弱性的压力测试情景：
- 模拟反映衰退风险或竞争压力的收入下降
- 假设压力下的利润率压缩（销量去杠杆、定价压力）
- 测试契约合规性和流动性
- 评估下行保护
- 记录正在压力测试的关键风险

**情景的记录要求：**
创建假设时间表显示：
- 每个情景的关键假设（收入增长、利润率、资本支出%）
- 每个调整的理由
- 关键变量的敏感性分析
- 历史预测准确性（如有）
- 与行业基准的比较

### 第五阶段：质量控制和验证

**步骤 5.1：数据准确性检查**
验证：
- 每个数字追溯到来源（检查 spot 样本，引用文档/URL/服务器）
- 所有计算都是基于公式的（无硬编码值）
- 小计和总数在数学上正确
- 年份显示无逗号（2024 而非 2,024）
- 无公式错误：#REF!、#VALUE!、#DIV/0!、#N/A

**步骤 5.2：格式一致性检查**
验证：
- 财务数据格式中有 $ 符号
- 运营数据没有 $ 符号
- 百分比显示为 %（15.0% 而非 0.15）
- 负数对财务数据使用括号
- 标题粗体且左对齐
- 数字右对齐
- 年份是文本格式

**步骤 5.3：结构和完整性检查**
确认：
- 所有必需工作表存在并正确排序
- 执行摘要简洁（适合一页）
- 所有关键指标全面捕获
- 从摘要到细节的逻辑流程
- 每个工作表中适当的粒度级别
- 无缺失数据或不完整部分

**步骤 5.4：专业呈现检查**
审查：
- 最小边框（仅用于结构）
- 一致缩进（子项目 2 空格）
- 适当的会计下划线（单线和双线）
- 整体干净、专业的外观
- 适当的列宽（不要太窄或太宽）

**步骤 5.5：文档和假设检查**
确保：
- 所有归一化调整都有记录的理由
- 包括源引用（文档页码、URL 或数据源引用）
- 假设清晰陈述且合理
- 执行摘要准确且有影响力
- 文件名包括公司名称和日期

### 第六阶段：最终交付

**步骤 6.1：创建执行摘要**
撰写简洁、有影响力的摘要，包括：
- 公司概述：商业模式、产品/服务、地理（2-3 句）
- 关键财务指标：收入、EBITDA、增长率（表格形式）
- 投资亮点：3-5 个关键优势或机会
- 值得注意的风险或考虑事项（简要）
- 如适用的交易背景

**步骤 6.2：最终文件准备**
- 使用适当的命名保存工作簿：CompanyName_DataPack_YYYY-MM-DD.xlsx

## 归一化模式

### EBITDA 的常见调整

**1. 重组费用**
- 如果真正非经常性则加回（设施关闭、一次性遣散费）
- 如果公司每年重组则不要加回
- 记录具体性质和非经常性的理由
- 示例："2023 年重组：300 万美元设施关闭，记录在源材料中，一次性事件"

**2. 股票薪酬**
- 行业标准：为私募股权分析加回
- 视为非现金运营费用
- 在所有期间保持一致
- 如果异常高或包括一次性授予则注意

**3. 收购相关费用**
- 加回交易费用、整合成本
- 按类型记录具体金额
- 不要加回持续的整合投资
- 引用每个调整的来源

**4. 法律和解和诉讼**
- 如果真正孤立事件则加回
- 评估复发风险（一次和解 vs. 诉讼模式）
- 记录和解性质
- 考虑这是否是正常业务过程

**5. 资产出售或减值**
- 从经营 EBITDA 中排除资产出售的损益
- 如果真正非经常性则移除减值费用
- 记录出售/减值的资产及原因
- 如果资产产生经营收入则调整收入

**6. 关联方调整**
- 归一化高于市场的关联方费用（租金、管理费）
- 调整到市场利率并有支持文档
- 移除通过业务运行的个人费用
- 记录市场利率比较

### 保守 vs. 激进归一化

**管理层情景：**
- 包括管理层提出的所有调整
- 接受公司对"非经常性"的定义
- 更激进的 EBITDA 调整
- 用于理解管理层的观点

**基准情景（投资决策建议）：**
- 仅清楚非经常性项目
- 对重复的"一次性"费用应用更高审查
- 排除投机性调整
- 更为保守，更能向投资委员会论证

## 行业特定适应

### 技术/SaaS
需要捕获的关键指标：
- ARR（年度经常性收入）和 MRR
- 按队列划分的客户数量
- CAC（客户获取成本）和 LTV（生命周期价值）
- 流失率（总流失和净流失）
- 净收入留存
- 40 法则（增长% + EBITDA 利润率%）
- 魔力数字（销售效率）

格式说明：ARR 是货币（$），客户数量是数字（无 $），比率是 %

### 制造/工业
需要捕获的关键指标：
- 生产产能和产能利用率 %
- 按产品线划分的产量
- 存货周转率
- 按产品线的毛利率
- 订单积压

格式说明：单位、产能是数字（无 $），利用率是 %，收入/成本是货币

### 房地产/酒店业
需要捕获的关键指标：
- 物业/房间/平方英尺
- 入住率 %
- ADR（平均每日房价）- 货币格式
- RevPAR（每可用客房收入）- 货币格式
- NOI（净营业收入）- 货币格式
- 资本化率 %
- FF&E 储备

格式说明：房间/平方英尺是数字，入住率是 %，ADR/RevPAR 是货币

### 医疗保健/服务
需要捕获的关键指标：
- 地点/设施
- 提供者/员工
- 患者/就诊次数（量指标）
- 每次就诊收入 - 货币
- 支付方组合 %
- 同店增长 %

格式说明：地点/就诊次数是数字，每次就诊收入是货币，比率是 %
