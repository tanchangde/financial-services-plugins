## 工作流程

### 1. 澄清需求
- **询问用户**：单页还是多页（3-4 页）？
- **询问用户**：任何特定的重点领域或主题？
- **仅在用户确认后**，进行研究

### 2. 研究与规划
**数据来源：**
- **主要**：公司申报文件（BamSEC、SEC EDGAR - "第 1 项 业务描述"、MD&A）、投资者演示文稿、公司网站
- **市场数据**：Bloomberg、FactSet、CapIQ（价格、股数、市值、净债务、EV、所有权）
- **估计**：FactSet/CapIQ 共识用于 NTM 收入、EBITDA、EPS
- **新闻**：过去 90 天的新闻稿、并购活动、业绩指引调整

**必需指标：**
- **财务**：收入、EBITDA（息税折旧摊销前利润）、利润率（%）、EPS、FCF 用于±3 年
- **估值**：市值、EV、EV/收入、EV/EBITDA、P/E 倍数
- **增长**：同比增长率（%）
- **所有权**：前 5 大股东及所有权%
- **细分**：产品组合和/或地理组合（% 细分）

**归一化：**
- 将所有金额转换为一致的货币
- 一致缩放（整个$mm 或$bn，不混合）

**构建之前：**
- 打印大纲到聊天，每项 4-5 个要点（实际数字，无占位符）
- 打印风格选择：字体、颜色（hex 代码）、每个数据集的图表类型
- 获得用户一致："这个大纲和视觉策略是否符合您的愿景？"

### 3. 逐页创建
**关键：您必须一次创建一页，并在继续下一页之前获得用户批准。**

**对于每张幻灯片：**
1. 仅用 PptxGenJS 创建这一页
2. **强制：转换为图像进行审查** - 您必须将幻灯片转换为图像以便直观验证：
   ```bash
   soffice --headless --convert-to pdf presentation.pptx
   pdftoppm -jpeg -r 150 -f 1 -l 1 presentation.pdf slide
   ```
3. **强制视觉审查**：在继续之前必须仔细检查渲染的幻灯片图像：
   - **文本重叠检查**：扫描每个文本元素 - 是否有任何标签、项目符号或标题相互碰撞？
   - **文本截断检查**：是否有任何文本在边界处被截断？所有单词是否完全可见？
   - **图表边界检查**：图表是否保持在容器内？所有轴标签是否完全可见？
   - **象限完整性**：一个象限中的内容是否溢出至相邻象限？
4. **如果检测到任何重叠或截断**：立即使用以下策略按顺序修复：
   - **第一**：减小字体大小（降低 1-2pt）
   - **第二**：缩短文本（缩写、删除不太关键的信息）
   - **第三**：调整元素位置或容器大小
   - **重新渲染并再次验证** - 在所有文本完全适合之前不要继续
5. 向用户显示幻灯片图像和下载链接
6. **停止并等待明确的用户批准**，然后再创建下一张幻灯片。在用户确认之前不要继续。

**您必须在每页上检查这些特定问题：**
- 表格行与下方的文本碰撞
- 图表 x 轴标签在底部被截断
- 长项目符号换行到相邻内容中
- 象限内容渗入相邻象限
- 标题文本与下方内容重叠
- 图例文本与图表元素重叠
- 页脚/来源文本与主要内容碰撞

## 幻灯片格式要求

### 信息密度至关重要

**#1 目标是最大信息密度。** 繁忙的高管应能在 30 秒内把握公司全貌。填充每个象限到容量。

**每象限目标：**
- **公司概述**：最少 6-8 个要点（总部、成立、员工、CEO/CFO、市值、股票代码、行业、关键统计）
- **业务与定位**：6-8 个要点（收入驱动因素、产品、市场份额%、竞争护城河、客户数量、地理组合）
- **关键财务**：8-10 行的表格或图表 + 4-5 个关键指标（收入、EBITDA、利润率、EPS、FCF、增长率、估值倍数）
- **第四象限**：5-7 个要点（所有权%、近期 M&A、发展、催化剂）

**信息打包技术：**
- 组合相关事实："总部：奥斯汀，德克萨斯州；成立于 2003 年；14 万员工"
- 始终包括数字："500 亿美元收入"而非"大收入"
- 添加背景："EBITDA 利润率：25%（vs. 18% 行业平均）"
- 包括同比变化："收入：1.25 亿美元（+28% 同比）"
- 使用百分比："企业：62% 收入"

**如果象限看起来稀疏，添加更多：**
- 细分分解与%
- 地理收入细分
- 客户集中度（前 10= X%）
- 近期合同获胜与$值
- 指引 vs. 共识
- 内部所有权%

**行间距 - 每个部分使用单个文本框：**
（代码示例保持与英文版本相同）

**关键间距原则：**
- 将标题 + 项目符号放在同一文本框中（无单独的标题文本框）
- 在标题后使用 `space_after = Pt(6)`，在项目符号之间使用 `Pt(3)`
- 不要硬编码间隙 - 让段落间距自然处理
- 如果内容溢出，减少字体 1pt 而不是删除内容

- **3-4 密集的幻灯片** - 使用象限、列、表格、图表
- **所有正文文本使用项目符号** - 永远不要段落。**每个部分使用一个文本框，所有项目符号在里面** - 不要为每个项目符号创建单独的文本框。使用 PptxGenJS 项目符号格式：
  ```javascript
  // 正确：单个文本框带项目符号列表 - 每个数组项成为一个项目符号
  // 在左上象限定位（公司概述）- 标题和强调条之后
  slide.addText(
    [
      { text: '总部：奥斯汀，德克萨斯州；成立于 2003 年', options: { bullet: { indent: 10 }, breakLine: true } },
      { text: '员工：全球 140,000+ 跨越 6 大洲', options: { bullet: { indent: 10 }, breakLine: true } },
      { text: 'CEO: Elon Musk; CFO: Vaibhav Taneja', options: { bullet: { indent: 10 }, breakLine: true } },
      { text: '市值：8500 亿美元（全球第 6）', options: { bullet: { indent: 10 }, breakLine: true } },
      { text: '细分：汽车（85%）、能源（10%）、服务（5%）', options: { bullet: { indent: 10 } } }
    ],
    { x: 0.45, y: 0.95, w: 4.5, h: 2.6, fontSize: 11, fontFace: 'Arial', valign: 'top', paraSpaceAfter: 6 }
  );

  // 错误：每个项目符号多个单独的文本框 - 导致对齐问题
  // slide.addText('总部：奥斯汀', { x: 0.5, y: 1.0, bullet: true });
  ```

  **项目符号格式提示：**
  - `bullet: { indent: 10 }` - 控制项目符号缩进（越小越紧）
  - `paraSpaceAfter: 6` - 每个段落后的空间（点）
  - 在每个项目符号中打包多个相关事实（例如"总部：奥斯汀；成立于：2003 年"）
  - 包括具体数字和百分比以增加信息密度
- **标题大小写**用于标题（非全大写），左对齐
- **一致字体** everywhere 包括表格
- **公司的品牌颜色** - 您必须在创建幻灯片之前通过网络搜索研究实际品牌颜色。不要猜测或假设颜色。
- **如果提供品牌指南则遵循**

## 第一页布局

必须通过"30 秒理解测试"为忙碌的高管。

### 幻灯片设置（关键）
**使用 4:3 纵横比**（标准 IB 路演格式）：
```javascript
const pptx = new pptxgen();
pptx.layout = 'LAYOUT_4x3';  // 10" 宽 × 7.5" 高 - 必须使用这个
```

### 幻灯片坐标系统
PptxGenJS 使用英寸。4:3 幻灯片 = **10" 宽 × 7.5" 高**。
- **x**：从左边缘的水平位置（0 = 左，10 = 右）
- **y**：从上边缘的垂直位置（0 = 顶，7.5 = 底）
- **内容必须保持在边界内** - 在所有边上留下 0.3" 边距

### 第一页定位（英寸）
（图表保持与英文版本相同）

### 标题部分（y=0.2）
**公司名称（股票代码）** - 示例：`Tesla, Inc. (TSLA)`

### 4 象限布局（y=0.6 到 y=7.2）

| 象限 | 位置 | 内容 |
|----------|----------|---------|
| **1** | x=0.3, y=0.6, w=4.7, h=3.0 | **公司概述**：总部、成立、关键统计、业务摘要（4-5 个要点） |
| **2** | x=5.0, y=0.6, w=4.7, h=3.0 | **业务与定位**：收入驱动因素、产品/服务、竞争地位、增长驱动因素（4-5 个要点） |
| **3** | x=0.3, y=3.7, w=4.7, h=3.5 | **关键财务**：收入、EBITDA、利润率、EPS、FCF + 估值（市值、EV、倍数） — **表格或图表，不同时使用** |
| **4** | x=5.0, y=3.7, w=4.7, h=3.5 | **对于上市公司**：1 年股价图表 + 前股东。**对于私有公司**：近期发展或所有权/M&A 历史 |

### 字体大小 - 使用这些精确值
| 元素 | 大小 | 注意 |
|---------|------|-------|
| 幻灯片标题 | 24pt | 粗体，公司品牌颜色 |
| 象限标题 | 14pt | 粗体，带装饰条 |
| 正文/项目符号文本 | 11pt | 常规权重 |
| 表格文本 | 10pt | 对密集表格使用 9pt |
| 图表标签 | 9pt | 保持标签简短 |
| 来源/页脚 | 8pt | 幻灯片底部 |

**关键：如果文本溢出，减少字体 1pt 并重新渲染。**
