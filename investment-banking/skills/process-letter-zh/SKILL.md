# 流程函件

## 工作流程

### 第一步：确定函件类型

- **初始流程函件**：与 teaser/CIM 一起发送，概述流程和 IOI 要求
- **IOI 指示**：第一轮兴趣指示的具体要求
- **第二轮/最终投标函件**：尽职调查后提交约束性报价的指示
- **管理层会议邀请**：现场管理层演示的后勤安排

### 第二步：初始流程函件 / IOI 指示

**页眉：**
- 日期、交易代号
- "机密"
- 发送对象：潜在买家

**部分：**

1. **介绍**：机会和卖方目标的简要概述
2. **流程概述**：时间表、关键日期、预期轮次数量
3. **IOI 要求**：初始指示中包括的内容：
   - 建议估值范围（企业价值）
   - 对价形式（现金、股票、盈利支付计划 (earnout)、股权滚动 (rollover)）
   - 融资来源和确定性
   - 关键尽职调查要求
   - 指示性关闭时间表
   - 任何条件或或有事项
   - 买方的简要描述和战略理由
4. **提交细节**：发送地点、截止日期（日期和时间）、格式
5. **保密提醒**：参考 NDA（保密协议）、数据室访问
6. **联系信息**：银行家联系人以供询问

### 第三步：最终投标 / 第二轮函件

除 IOI 之外的额外要求：

1. **购买协议修订意见**：提供草案 SPA/APA（股权/资产购买协议）并请求修订意见
2. **详细融资承诺**：需要承诺的融资函
3. **剩余尽职调查项目**：说明预期哪些确认性尽职调查
4. **排他性条款**：任何排他性期限的持续时间和条件
5. **监管分析**：反垄断申报要求和时间表
6. **关键人员条款**：雇佣协议、薪酬、rollover 股权
7. **约束性与非约束性**：澄清在此阶段哪些是约束性的
8. **评估标准**：投标将如何评估（价格、确定性、速度、匹配度）

### 第四步：管理层会议邀请

1. **后勤**：日期、时间、地点（或视频链接）、持续时间
2. **出席者**：公司谁将演示、买家谁应该出席
3. **议程**：典型的管理层演示议程（概述、财务、运营、增长、问答）
4. **基本规则**：无录音、保密、问题格式
5. **材料**：将分发什么（演示幻灯片、数据室访问）
6. **后续**：会议后提交额外问题的流程

### 第五步：输出

- Word 文档（.docx）具有专业函件格式
- 公司函头占位符
- 用于客户审查的跟踪更改版本

## 重要说明

- 流程函件奠定整个交易的基调 — 清晰、专业、有条理
- 截止日期应明确且合理 — 通常 IOI 为 2-3 周，最终投标为 3-4 周
- 始终包括评估标准 — 买方希望了解评标标准
- 与法律部门协调函件中的任何陈述或承诺
- 发送前客户应审查和批准 — 客户可能希望调整措辞语气或交易条款
- 保留每份函件的接收者和时间记录 — 这成为流程跟踪器
